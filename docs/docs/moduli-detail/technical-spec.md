# Техническое задание (клиент bbplay)

## Цель

Мобильное приложение **BlackBears Play**: доступ к сети клубов, **бронирование ПК**, просмотр **новостей**, **профиля** и **баланса**, **чат поддержки** на основании загруженной **базы знаний** (без внешних LLM), **напоминания** о бронях и сценарии **обратной связи** после визита.

## Внешние системы

- **Vibe-прокси** BlackBears — основной HTTP-контур (`EXPO_PUBLIC_API_BASE_URL`).
- **iCafe API** — либо через прокси, либо прямой Cloud (специальные env).
- **VK** (опционально) — новости.
- **EAS** — сборка; push — FCM / APNs при настройке.

## Функциональные требования (уровень продукта)

1. **Аккаунт:** регистрация, вход, при необходимости SMS, выход, сохранение сессии.
2. **Клубы:** список, детали, схема зала, статусы ПК, геосортировка.
3. **Бронь:** выбор слота, создание, отмена, отображение «сегодня», согласование с уведомлениями и при желании с календарём ОС.
4. **Платежи/баланс:** отображение и сценарии topup **в соответствии с** разрешениями API.
5. **Профиль:** настройки (тема, город, внешний вид), инсайты (списки/ WebView) по режиму `INSIGHTS_UI_MODE`.
6. **Новости:** открытие ленты по конфигу.
7. **Поддержка:** ответы из локальной/загруженной базы знаний.
8. **Стабильность:** `AppErrorBoundary`, предсказуемые ошибки API.

## Нефункциональные

- **Платформы:** iOS, Android; Expo 51; TypeScript.
- **Производительность:** разумные `staleTime`, без лишнего `refetchOnWindowFocus` на RN.
- **Документация:** эта папка + `docs/api-spec.md` и связанные контракты.
- **Безопасность:** см. [submoduli/security.md](../submoduli/security.md).

## Граница ответственности

- Ценообразование, валидация брони «на стороне сервера» — **не** в клиенте.
- Открытие/закрытие путей на **прокси** — **не** в клиенте (403 / «Api not allowed»).

## Критерии готовности релизной сборки (для команды; дополняйте)

- Прохождение `npm run lint` и `npm run test:unit` в CI/локально.
- Smoke: вход → клубы → создание/отмена брони на **целевом** стенде, пополнение (если в скоупе) по согласованным правам API.
- Проверка `EXPO_PUBLIC_*` **в EAS** для данного профиля совпадает с ожидаемым бэкендом.
- Регрессия пуш-напоминаний на **релизной** конфигурации, не в Expo Go (если пуши в релизе).
