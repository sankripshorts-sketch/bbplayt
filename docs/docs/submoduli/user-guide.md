# Руководство пользователя (обзор)

Описание сценариев **с точки зрения экранов приложения**. Контракты полей API — в `docs/api-spec.md`, `icafe-api.md`, `API_VIBE_LOGIC.md`.

## Нижние вкладки

После входа доступны пять разделов (см. [modules-map.md](../modules-map.md)):

### Профиль (таб «Профиль»)

- **Домашний экран профиля:** баланс, быстрые действия, переход к настройкам, пополнению, промо-ленте (по продукту).
- **Новости** — в стеке профиля: лента (часто VK / WebView), настраивается `EXPO_PUBLIC_VK_*`.
- **Настройки:** город, внешний вид (тема/шрифты по возможностям), напоминания о брони, редактирование профиля.
- **Деньги / аналитика:** движения по счёту, игровые сессии, аналитика, рейтинг — в зависимости от `EXPO_PUBLIC_INSIGHTS_UI_MODE` (заглушка vs **live**); при ошибке API — fallback URL.
- **Пополнение:** WebView/каталог; URL и тело запросов зависят от `EXPO_PUBLIC_TOP_*` и прав на прокси.

### Клубы

- Список клубов: фильтр/сортировка, **сортировка по расстоянию** при разрешении геолокации.
- Карточка клуба: схема зала, статусы ПК, ссылки (карты, отзыв — `EXPO_PUBLIC_JOB_REVIEW_URL` и данные API).

### Еда

- Каталог/меню в объёме, который отдаёт продукт и API (см. `FoodContext`, `FoodScreen`).

### Бронь

- Выбор клуба, даты, интервала, тарифа, зоны, **конкретного ПК** (схема и/или список).
- Создание брони — **подписанный** POST; при успехе — баннер «сегодня», опционально напоминание и запись в календарь.
- **Мои брони**, отмена, поиск **ближайшего окна** (если реализовано в билде).
- Повтор сценария из уведомления: deep link + префилл (см. `BookingPrefillParams` в `navigation/types.ts`).

### Помощь (чат)

- Ответы строго из **загруженной базы знаний** (`assets/knowledge.json` и/или URL в конфиге). Облачные LLM **не** вызываются.

## Авторизация (до появления табов)

- **Вход** и **регистрация** в стеке `AuthNavigator` до получения валидной сессии.
- **SMS** — если включено бэкендом; пути `request-sms` / альтернативы в `.env`.
- См. подробнее [account-login.md](account-login.md).

## Уведомления

- **Напоминания** о предстоящей брони (локальные, `expo-notifications`).
- **Запрос отзыва** после визита (логика в `notifications/` + контекст).

## Ограничения

- Без сети полноценного офлайна нет: часть кэша React Query персистится выборочно.
- 403 / «Api not allowed» — не баг кнопки, а **политика прокси** на сервере.
