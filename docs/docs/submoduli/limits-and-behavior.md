# Ограничения и ожидаемое поведение

## Сеть

- Без сети после холодного старта **полного** офлайна нет: только часть кэша React Query переживает перезапуск.
- Повторные запросы — по политике `retry` в `queryClient`.

## Бэкенд

- Ответы и доступность эндпоинтов **определяются прокси** (403, пустые списки).
- Поведение брони (минимальный слот, прошлое время) валидируется сервером; клиент дублирует UX-проверки, но не заменяет API.

## Платформа

- **Expo Go** — не все нативные сценарии 1:1 с production build; для пушей и т.д. используйте dev client / EAS.
- **Web** — вторичный таргет; ориентир — iOS/Android.

## Чат

- Только **локальная** база знаний: длинные и нестандартные вопросы могут не находить ответа.

## Гео

- Сортировка по расстоянию требует разрешения на геолокацию; при отказе — fallback на сортировку по умолчанию.

## Схема зала

- Координаты из API могут требовать **подгонки** env (`HALL_MAP_*`); визуальное пересечение зон/ПК — не баг сети, а маппинг.
