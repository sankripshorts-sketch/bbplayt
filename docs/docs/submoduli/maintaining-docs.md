# Сопровождение этой документации

## Структура

- **`docs/ARCHITECTURE.md`** — единая «витрина» стека; при крупных изменениях (новый провайдер, смена персиста) — обновляйте и при необходимости **дубли** в `docs/docs/modules/architecture.md`.
- **`docs/docs/README.md`** — индекс подпапки: новый раздел → строка в таблице + ссылка в «Сквозные темы».
- **`docs/docs/modules-map.md`** — при добавлении **нового** верхнего каталога в `src/` обновите таблицу.
- **Контракты API** — `docs/api-spec.md` и соседние файлы; не копируйте длинные тела в `submoduli`, держите **ссылки**.

## Согласованность

- Имена env — как в [`.env.example`](../../../.env.example).
- Поведение React Query — как в `src/query/queryClient.ts` (какие ключи персистятся).
- `App.tsx` — источник истины по стеку провайдеров.
- Практические ссылки **куда смотреть в коде** — [../developer-guide.md](../developer-guide.md) (дополняйте при появлении устойчивых «рецептов»).

## Что не делать

- Не оставляйте в bbplay-репо длинные куски, скопированные из **telegram-bot** / **gbrain**, если они не относятся к коду (удаляйте дубликаты, заменяйте ссылками на актуальные внутри репо).

## Промпты

Скрипты `export:cursor-prompts` / `json-prompts-to-md` в корне; артефакты `PROMPTS_ALL.md` — отдельно от архитектурной доки.

См. также: [../README.md](../README.md), [related-documentation.md](related-documentation.md).
