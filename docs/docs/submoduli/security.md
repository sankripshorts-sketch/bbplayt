# Безопасность (клиент)

## Поверхность риска

- Клиент **всегда** обратим: строки `EXPO_PUBLIC_*` в бандле. Не кладите туда настоящие серверные секреты, если они не рассчитаны на публичное знание (как в эталоне Android с договорной подписью).
- `SecureStore` снижает риск утечек по сравнению с обычным хранилищем, но не защищённый от jailbreak/root.
- **Логи** в проде не должны печатать токены и PII.

## Сеть

- Предпочтительно **HTTPS** (в проде — по умолчанию). MITM — риск на устройстве с корневыми сертификатами.
- Сетевая политика прокси (списки путей) — на стороне сервера.

## Рекомендации для разработчиков

- Регулярно обновляйте Expo / RN при патчах безопасности.
- `patch-package`: ревью каждого патча; не вливать неаудированные куски.

## Уведомления

- Планирование на точное время; не пишите в payload чувствительные данные, которых нет в публичных бронях.
